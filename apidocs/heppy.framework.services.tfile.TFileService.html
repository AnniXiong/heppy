heppy.framework.services.tfile.TFileService(Service) class documentation

Part of heppy.framework.services.tfile (View In Hierarchy) 
TFile service.
The file attribute is a TFile that can be used in several analyzers.
The file is closed when the service stops.

Example configuration:

output_rootfile = cfg.Service(
  TFileService,
  'myhists',
  fname='histograms.root',
  option='recreate'
)
Method __init__ cfg must contain: - fname: file name - option: TFile options, e.g. recreate
Method stop Stop the service. Called by the looper, not by the user.

  • fname: file name

    outdir is the output directory for the TFile

    comp is a dummy parameter here. It is needed because the looper creates services and analyzers in the same way, providing the configuration (cfg), the component currently processed (comp), and the output directory.

    Other implementations of the TFileService could make use of the component information, eg. the component name.
